Final Year Project Proposal

VicPi Security

A Smart Home and Office Surveillance and Access Control System

Student Name Victor Mulinge (vicpod)
Submission Date April 2026
Hardware Platform Raspberry Pi 4 Model B (4GB)
Category IoT / Embedded Systems / Cybersecurity

1. Introduction

This project is a smart surveillance and access control system built to provide high-quality security at a price point that actually works for people. By using affordable hardware like Raspberry Pi or ESP32 controllers, solenoid locks, and even repurposed webcams, it delivers professional-grade protection without the massive market markup. Imagine being able to leave your heavy luggage in a secure locker room in the CBD, knowing it's protected by RFID or facial recognition, while you go about your business freely? Whether it's for urban locker hubs, home security, or e-commerce delivery points, this project aims to make safety and convenience accessible to everyone. It bridges the gap between expensive proprietary systems and local needs, offering a remotely-accessible platform that proves security doesn't have to be a luxury.

The rapid growth of smart technology and the Internet of Things (IoT) has created new opportunities for affordable, intelligent security solutions. Traditional surveillance and access control systems — relying on passive CCTV cameras, mechanical locks, and dedicated DVR hardware — are increasingly inadequate for modern security requirements. They lack intelligence, generate enormous volumes of unreviewed footage, cannot respond autonomously to security events, and require significant financial investment for installation and maintenance.

VicPi Security addresses these limitations by delivering a fully integrated smart surveillance and access control system built on the Raspberry Pi 4 — a low-cost, high-performance single-board computer. The system combines live video surveillance, AI-powered facial recognition, multi-method physical access control, real-time push notifications, and a remotely accessible web dashboard into a single unified platform. By leveraging open-source software and commodity hardware, VicPi Security achieves capabilities typically reserved for enterprise-grade systems, at a fraction of the cost.

2. Problem Statement

Current surveillance and access control solutions in the market present several critical shortcomings that limit their practical utility for home and small office environments:

  • Traditional CCTV systems are passive — they record continuously without intelligence, requiring manual review of footage after incidents rather than preventing them.
  • Standalone access control systems (keypads, RFID readers) operate independently from surveillance cameras, providing no visual verification of who is attempting access.
  • Commercial integrated systems that combine surveillance and access control are prohibitively expensive, typically requiring professional installation and ongoing licensing fees in the thousands of dollars.
  • Existing solutions do not provide real-time alerts with visual evidence — property owners are notified of incidents only after reviewing recorded footage.
  • Remote monitoring and control capabilities in affordable systems are limited, preventing property owners from managing access from outside the premises.
  • Most systems fail gracefully — a power cut disables the entire security infrastructure without automatic recovery.

3. Proposed Solution

VicPi Security proposes the design and implementation of a multi-layered smart security system built on the Raspberry Pi 4 platform. The system integrates the following capabilities into a single unified platform:

3.1 Surveillance

A Pi Camera Module streams live MJPEG video to a web-based dashboard accessible from any device on the local network or remotely via Cloudflare Tunnel. The camera doubles as the sensor for the face recognition engine, eliminating the need for separate hardware.

3.2 Multi-Method Access Control

Three independent access methods are supported, providing redundancy and flexibility. Face recognition using the OpenCV LBPH algorithm identifies enrolled individuals and automatically triggers door unlock. A 4×4 membrane keypad allows PIN-based entry for situations where face recognition is impractical. A remote unlock button on the web dashboard enables administrators to grant access from anywhere in the world.

3.3 Physical Hardware Integration

A relay module connected to GPIO 23 controls an electromagnetic door lock via a DC buck converter. Visual feedback is provided through green and red LEDs indicating access granted and denied respectively. An active buzzer driven through a 2N3904 NPN transistor provides audio feedback — two tones for doorbell, three short beeps for access denied. A push button doorbell simultaneously rings the buzzer, triggers face recognition, takes a visitor snapshot, and dispatches a Telegram notification.

3.4 Real-Time Notifications

The Telegram Bot API is integrated to deliver push notifications with camera snapshots to the administrator's phone within seconds of any access event — doorbell press, successful access, or failed access attempt. This transforms the system from a passive recorder into an active security alert system.

3.5 Web Dashboard

A Progressive Web Application (PWA) dashboard built with HTML5, CSS3, and JavaScript provides a professional monitoring interface featuring a live camera feed, real-time access log with WebSocket updates, face enrolment interface, access control buttons, event statistics, and dark-themed industrial UI. The PWA is installable on mobile devices as a native-like app without App Store submission.

3.6 Remote Access

Cloudflare Tunnel creates a secure encrypted connection between the Pi and Cloudflare's global edge network, exposing the dashboard and API via a public HTTPS URL accessible from anywhere in the world without firewall configuration or port forwarding.

3.7 System Resilience

Both the backend API server and dashboard are configured as systemd services, ensuring automatic startup on boot and automatic recovery from crashes. A static IP address eliminates URL changes across reboots, and the SQLite database persists all access logs to disk for historical review.

4. Project Objectives

4.1 Primary Objectives

  • To design and implement a multi-method access control system supporting face recognition, keypad PIN entry, and remote app unlock on a Raspberry Pi 4 platform.
  • To integrate live video surveillance with access control in a unified system, eliminating the need for separate surveillance and access hardware.
  • To develop a real-time web dashboard accessible from local and remote networks that provides live camera monitoring, event logging, and access control capabilities.
  • To implement automated push notification delivery with camera snapshots via Telegram for all security events.
  • To deploy the system with automatic boot recovery ensuring continuous operation without manual intervention after power interruptions.

4.2 Secondary Objectives

  • To evaluate the performance of the OpenCV LBPH face recognition algorithm in a real-world embedded deployment environment.
  • To demonstrate the viability of low-cost commodity hardware as a platform for intelligent security systems traditionally requiring expensive proprietary equipment.
  • To document the design, implementation, and testing process in a manner suitable for replication and further development.

6. System Architecture

The VicPi Security system is structured around three architectural layers that communicate through well-defined interfaces:

  • Hardware Layer: The Raspberry Pi 4 and all peripheral components (camera, keypad, relay, LEDs, buzzer, door lock) form the physical sensing and actuation layer. GPIO pins provide the electrical interface between the Pi and all peripherals.
  • Backend Layer: A Python FastAPI server running on the Pi manages all business logic — face recognition, PIN verification, GPIO control, database logging, WebSocket broadcasting, and Telegram notification dispatch. The server exposes a REST API on port 8000 and serves the dashboard on port 8080.
  • Presentation Layer: The Progressive Web Application dashboard communicates with the backend via REST API calls and WebSocket connections, delivering live video, real-time event updates, and access control functionality to any connected device.

Communication between layers uses standard protocols: HTTP/REST for API calls, WebSocket for real-time event streaming, MJPEG over HTTP for video streaming, and HTTPS via Cloudflare Tunnel for encrypted remote access.

7. Expected Outcomes and Impact

7.1 Technical Outcomes

  • A fully functional integrated surveillance and access control system demonstrating real-time face recognition, multi-method door access, and automated security alerting on commodity embedded hardware.
  • An LBPH face recognition implementation achieving greater than 85% accuracy in controlled indoor environments on a resource-constrained ARM processor.
  • A remotely accessible web dashboard with sub-second real-time event updates via WebSocket, demonstrating practical IoT system design principles.
  • A documented, replicable system architecture that can serve as a foundation for further research and commercial development.

7.2 Market and Social Impact

VicPi Security demonstrates that intelligent, integrated security technology is achievable at a component cost of approximately KES 15,000–25,000 — compared to tens of thousands of dollars for equivalent commercial enterprise systems. This democratisation of smart security technology has significant implications for small businesses, educational institutions, residential properties, and community facilities in markets where commercial enterprise security solutions are financially inaccessible. The project also demonstrates the broader applicability of single-board computer platforms for IoT applications beyond hobbyist use, contributing to the growing body of evidence supporting embedded Linux systems as viable platforms for production-grade intelligent systems.

7.3 Academic Contribution

  • A comparative evaluation of face recognition algorithms (Eigenfaces, Fisherfaces, LBPH, deep learning) in the context of embedded real-time access control deployment.
  • Practical documentation of the challenges of deploying Python machine learning libraries on ARM64 hardware with Python 3.13, and the solutions employed.
  • A demonstrated integration pattern for combining computer vision, GPIO hardware control, REST APIs, WebSocket, and cloud tunnelling in a unified IoT application.

11. Conclusion

VicPi Security represents a meaningful contribution to the field of affordable intelligent security systems. By integrating computer vision, IoT hardware control, web application development, cloud networking, and real-time communication technologies into a single coherent platform, the project demonstrates the transformative potential of modern single-board computer hardware when combined with the rich ecosystem of open-source software libraries available in Python.

The system directly addresses a genuine market gap — the absence of affordable, integrated, intelligent security solutions for home and small office environments — and does so using commodity hardware costing a fraction of commercial equivalents. The technical challenges encountered and solved during development, particularly the Python 3.13 compatibility issues with deep learning frameworks and the Pi Camera integration with newer Pi OS releases, contribute practical knowledge to the embedded systems and IoT development communities.

The project demonstrates competence across multiple engineering disciplines including embedded systems hardware design, Python software engineering, computer vision algorithm deployment, REST API design, real-time web application development, database management, network security, and cloud infrastructure — making it a comprehensive showcase of final year engineering capability.
